Subject: Implement Pydantic schema for LLM-powered SQL query builder

Message:
This commit implements a production-ready Pydantic schema designed for
natural language to SQL translation using LLM structured outputs.

Key Features:
- No recursive types (LLM-compatible)
- Discriminated unions for expression types
- Explicit depth control (L0/L1 WHERE clauses)
- Enum-based schema (tables, columns, operators, functions)
- Comprehensive SQL support (JOINs, subqueries, window functions, CASE)

Architecture:
- Clause-based structure mirroring SQL
- 34 Pydantic models
- 6 expression types (Column, Arithmetic, Aggregate, Window, Case)
- Explicit nesting limits (no infinite recursion)

Testing:
- 53 unit tests (100% passing)
- Full SQL translation validation
- Google Vertex AI compatibility verified
- All pricing analyst use cases validated

Documentation:
- Comprehensive README
- Detailed Vertex AI findings and best practices
- 14 working examples (basic + pricing analyst queries)
- Implementation summary
